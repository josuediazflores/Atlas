ATLAS

An agent loop that knows when research is done.

Atlas runs an autonomous research-synthesis loop over interview sessions: it reads each transcript as it lands, measures how much is genuinely new, and tells the team the moment further interviews stop teaching anything.

I.The problem

Qualitative research has no stopping point.

A study plan says “recruit twelve to fifteen participants” because nobody knows the real number. So teams keep recruiting and keep interviewing — often well past the point where new sessions teach them anything new.

The question “have we heard enough?” is usually answered by feel, if it is answered at all. And every extra session that runs past that point costs real money.

i
Incentives

Every participant is paid, whether or not their session adds a single new theme.

ii
Recruiting

Sourcing, screening, and scheduling each additional participant carries its own fee.

iii
Researcher time

Moderating, note-taking, and synthesis hours accumulate on sessions that change nothing.

II.The insight

Saturation is a stopping condition you can verify.

Qualitative researchers have had the answer for decades. It is called saturation: you stop interviewing when new sessions stop yielding new themes.

What makes saturation interesting to an engineer is that it is verifiable. An autonomous loop needs a halting condition it can check — not a vibe, a measurement. Score each session for the novelty it adds; when that score stays under a threshold, the work is done. Saturation turns “enough research” into a condition a machine can test.

III.How the loop works

Five steps, repeated until a verdict.

Each pass through the loop consumes exactly one session and ends with a decision: continue, or halt with a verdict.

  1. i
    Fetch

    Pull the next session transcript through Great Question's MCP server — search_studies, list_repo_sessions, get_transcript.

  2. ii
    Extract

    An LLM reads the transcript and extracts themes, each anchored to the verbatim quotes that support it.

  3. iii
    Score novelty

    New themes are embedded and compared against the accumulated set. The distance between them becomes the session's novelty score.

  4. iv
    Verify

    A separate verifier model grades the extraction for faithfulness to the transcript. The worker never marks its own homework.

  5. v
    Decide

    Continue or halt, judged against the saturation threshold, the quality floor, and the budget cap — then return to the top for the next session.

IV.Four ways a run ends

Every ending is a verdict, not an error.

A run never simply stops. It concludes — and each conclusion tells the team something different to do next.

Verdict 1

Saturated

Novelty stayed below the threshold for k consecutive sessions. The themes have converged; new interviews are repeating old ones.

→ Stop recruiting
Verdict 2

Quality halt

The verifier's faithfulness score dropped below the floor. The synthesis can no longer be trusted, so the loop refuses to continue on top of it.

→ Review extraction before resuming
Verdict 3

Budget halt

Spend exceeded the cap set at the start of the run. The loop stops before the budget does — never the other way around.

→ Raise the cap or accept partial synthesis
Verdict 4

Not saturated

Sessions ran out while novelty was still above the threshold. The field is still teaching; the study genuinely needs more voices.

→ Keep recruiting

VI.Trust by design

Autonomy, bounded on every side.

An agent that spends money and writes findings has to be built defensively. Four constraints hold on every run, without exception.

№ 1

Every claim is grounded

No finding exists without a citable verbatim quote, and every quote is traceable to its source session and timestamp.

№ 2

The worker never grades itself

Extraction and verification are performed by separate models. The one that does the work is never the one that judges it.

№ 3

Writes wait for a human

Any write action against the research platform is held until a person explicitly approves it. Reading is autonomous; changing things is not.

№ 4

A hard budget cap

Every run starts with a spending ceiling. Crossing it halts the loop — a verdict, recorded and reported like any other.

VII.About this project

A working prototype, honestly framed.

Atlas is an engineering prototype, not a product. It was built to test one idea: that saturation — a concept qualitative researchers already trust — gives an autonomous agent loop a stopping condition it can actually verify. It runs against real studies through Great Question's MCP integration.
